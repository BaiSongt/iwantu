# iWantU 产品需求文档 PRD v1.0

## 1. 项目定位

iWantU 是面向人类用户与外部 AI Agent 的 AI 能力供需撮合、方案采购、POC 验证与交付协同平台。

平台同时支持两种模式：

- **传统人类模式**：企业/个人发布需求，AI公司、OPC团队、开发者发布产品、Agent、解决方案，双方通过平台完成联系、报价、POC与采购沟通。
- **AI自主模式**：外部 AI 可以机器可读方式浏览平台内容，并在授权范围内代表用户发布需求、生成需求草稿、代表供应商接单、生成方案、参与POC沟通。

核心闭环：

```text
发现AI能力 → 发布需求 → AI澄清 → 智能匹配 → 申请POC → 提交方案/报价 → 沟通确认 → 采购/交付
```

## 2. 用户角色

### 2.1 人类角色

| 角色 | 说明 | 核心行为 |
|---|---|---|
| 游客 | 未登录访问者 | 浏览公开产品、精选、公司、解决方案 |
| 需求方用户 | 企业/个人采购方 | 发布需求、申请POC、对比产品、联系供应商 |
| 供应商用户 | AI公司、开发者、OPC团队 | 发布产品、Agent、解决方案，接收线索 |
| OPC公司/团队 | 具备交付能力的服务团队 | 展示能力、接单、提交方案、参与POC |
| 平台运营 | iWantU运营方 | 内容审核、推荐配置、榜单运营、纠纷处理 |
| 平台管理员 | 系统管理员 | 权限、风控、数据、系统配置 |

### 2.2 AI角色

| AI角色 | 说明 | 权限边界 |
|---|---|---|
| 外部浏览AI | 读取公开内容 | 仅访问公开页面、公开API、llms.txt |
| 需求代理AI | 代表需求方提需 | 需用户授权；默认只能创建草稿 |
| 供应商接单AI | 代表供应商接单 | 需供应商授权；可生成方案草稿 |
| 平台撮合AI | iWantU内部AI | 需求解析、匹配推荐、风险提示 |
| POC评测AI | 自动评测POC结果 | 只访问授权POC数据 |
| 风控审核AI | 辅助审查内容与行为 | 不能单独执行处罚性动作 |

## 3. AI自主模式

### 3.1 AI可访问层级

| 层级 | 内容 | 是否登录 | 是否授权 |
|---|---|---|---|
| L0 公开内容 | 首页、精选、公开产品/公司/方案 | 否 | 否 |
| L1 机器可读内容 | JSON-LD、Markdown、llms.txt、公开API | 否 | 否 |
| L2 登录内容 | 需求详情、部分供应商资料、POC说明 | 是 | 视角色 |
| L3 授权内容 | 私有需求、报价、POC数据 | 是 | 必须授权 |
| L4 敏感内容 | 合同、付款、联系方式、私有附件 | 是 | 强授权+审计 |
| L5 执行动作 | 发布需求、提交方案、自动接单、发起POC | 是 | 明确动作授权 |

### 3.2 AI行为授权等级

| 等级 | 名称 | 说明 |
|---|---|---|
| A0 | 禁止AI代理 | AI只能浏览公开内容 |
| A1 | AI只读 | 可读取授权内容，不可操作 |
| A2 | AI草稿 | 可创建需求/方案/报价草稿，需人工提交 |
| A3 | AI半自动 | 可执行低风险动作，高风险需确认 |
| A4 | AI自主接单/提需 | 在预算、范围、规则内自动执行 |
| A5 | AI高风险授权 | 涉及合同、付款、敏感资料，必须二次确认 |

### 3.3 供应商AI接单设置

供应商后台必须提供：

- 是否允许AI自主接单：开/关
- 是否允许AI自动提交方案：不允许/草稿/自动提交
- 预算上限
- 可接行业范围
- 可接需求类型
- 最低匹配度
- 是否必须人工复核
- 是否允许AI读取历史案例
- 是否允许AI读取报价模板
- 是否允许AI参与站内沟通：只读/草稿/自动回复

### 3.4 需求方AI提需设置

需求方控制台必须提供：

- 是否允许AI代表我发布需求
- 发布模式：只生成草稿/提交后待审核/自动发布
- 是否匿名发布
- 是否允许AI上传附件
- 预算上限
- 是否允许AI发起POC
- 是否允许AI联系供应商

## 4. 全局功能需求

### 4.1 机器可读内容

所有关键对象页面需要同时输出：

- HTML：给人类用户浏览
- JSON-LD：给搜索引擎与AI理解
- Markdown：给外部AI快速读取
- OpenAPI：给授权系统调用
- MCP工具服务：给AI工具调用
- A2A Agent Card：给外部Agent发现平台能力

### 4.2 核心业务对象

```text
User / Organization / AIIdentity / Product / AgentProduct / CompanyProfile / Demand / Solution / POCProject / Proposal / Quote / MessageThread / PermissionPolicy / AuditLog / AgentAction
```

### 4.3 状态模型

需求状态：

```text
草稿 → 待澄清 → 待报价 → 方案征集中 → POC中 → 已成交 / 已关闭
```

产品状态：

```text
草稿 → 待审核 → 已上架 → 需补充资料 / 已下架 / 被风控
```

POC状态：

```text
未开始 → 需求确认 → 样例数据上传 → 供应商测试 → 结果评审 → 采购沟通 → 已完成 / 已终止
```

AI行为状态：

```text
待授权 → 已授权 → 执行中 → 需人工确认 → 已执行 / 被拒绝 / 已撤销 / 异常拦截
```

## 5. 页面级PRD

### 5.1 首页 Home

目标：平台总入口。

功能：平台定位、需求搜索框、发布需求入口、热门产品、最新需求、优质公司、精选内容入口。

AI增强：输出 `/index.md`、首页 JSON 摘要、平台导航结构、llms.txt核心入口。

权限：游客可浏览；AI只读公开内容；发布需求需登录。

### 5.2 精选推荐 Featured Store

目标：类似 App Store Today 的运营推荐页。

功能：横幅推荐、推荐产品、OPC公司、优秀团队、专题推荐、运营位管理。

AI增强：输出推荐理由、推荐对象类型、machine_summary。

权限：平台运营配置推荐；供应商不可直接修改推荐状态。

### 5.3 AI产品列表

目标：产品市场浏览与筛选。

功能：图片主导卡片、行业筛选、能力筛选、部署方式、价格模式、POC、私有化筛选。

AI增强：Product JSON-LD、产品列表API、Markdown列表。

权限：公开产品可读；未公开价格可登录可见；供应商可设置AI可读范围。

### 5.4 需求大厅

目标：供应商发现可承接需求。

功能：需求列表、预算/周期/行业筛选、查看详情、提交方案、收藏需求。

AI增强：供应商AI判断匹配度、生成方案草稿、AI接单控制。

权限：需求可匿名；附件和联系方式默认授权可见。

### 5.5 产品详情

目标：完成采购判断。

功能：产品主图、缩略图、产品参数、能力标签、申请POC、联系供应商、加入对比、详情图文、案例、附件、评价。

AI增强：Markdown详情、结构化参数、potentialAction、能力边界说明。

权限：报价资料、附件和敏感参数可设为授权可见。

### 5.6 智能匹配结果

目标：根据需求推荐产品、Agent、公司和方案。

功能：需求摘要、标签、推荐列表、匹配理由、风险提示、下一步建议。

AI增强：匹配置信度、推荐证据、可调用匹配API。

权限：匹配结果为需求方私有数据。

### 5.7 发布需求

目标：将模糊需求结构化。

功能：需求标题、行业、痛点、任务、现有系统、数据类型、预算、周期、POC、联系方式、AI澄清助手。

AI增强：AI代填、AI追问、生成需求书、创建需求草稿。

权限：AI默认只能生成草稿；涉及预算、附件、联系方式需确认。

### 5.8 发布产品

目标：供应商发布AI产品。

功能：产品名称、公司主体、一句话介绍、主图、详情图、类型、价格、行业、部署、富文本、案例、附件、提交审核。

AI增强：产品资料自动补全、摘要生成、完整度检查。

权限：AI发布默认草稿；提交审核需授权。

### 5.9 需求详情

目标：供应商判断是否接单。

功能：需求信息、预算、周期、数据、部署、附件、匹配建议、提交方案、申请沟通。

AI增强：供应商AI读取授权字段、生成方案草稿、风险提示。

权限：公开字段与授权字段分离。

### 5.10 需求方控制台

目标：采购方管理需求和POC。

功能：我的需求、匹配结果、收到方案、POC进度、收藏、对比、消息中心、AI授权设置。

AI增强：AI代理权限、AI行为记录、草稿审批、预算边界。

权限：组织成员访问；管理员设置组织级AI策略。

### 5.11 供应商后台

目标：供应商管理产品、线索、方案、POC。

功能：我的产品、需求线索、提交方案、POC项目、公司主页、认证资料、AI接单设置。

AI增强：自动筛选线索、生成方案、AI接单、报价模板。

权限：高风险AI动作必须授权与审计。

### 5.12 产品对比

目标：横向比较产品。

功能：核心能力、行业、部署、价格、POC、私有化、案例、评分、推荐结论。

AI增强：生成对比报告、采购建议、风险差异。

权限：对比列表属于用户私有数据。

### 5.13 AI Agent专区

目标：展示能力型Agent。

功能：Agent列表、任务目标、输入输出、工具调用、成功率、部署、价格、POC。

AI增强：Agent Card、外部AI发现、调用限制说明。

权限：能力公开，执行接口鉴权。

### 5.14 Agent详情

目标：说明Agent能做什么和不能做什么。

功能：任务目标、输入、输出、工具调用、成功率、风险边界、人工审核建议、POC。

AI增强：能力卡、可调用动作、失败边界。

权限：执行类Agent需授权。

### 5.15 AI公司与OPC列表

目标：展示AI公司、OPC团队、交付方。

功能：公司列表、横幅、认证、主营能力、行业经验、交付范围、案例、评分。

AI增强：按交付能力搜索，推荐OPC团队。

权限：公司公开资料可读；报价、合同、客户联系人不可公开。

### 5.16 OPC公司主页

目标：建立供应商可信档案。

功能：封面、Logo、认证标签、主营能力、项目、案例、产品、评分、响应率、联系方式。

AI增强：公司能力摘要、服务范围、可接单条件。

权限：认证信息平台维护；供应商设置AI可读范围。

### 5.17 解决方案专区

目标：按场景打包AI能力。

功能：方案列表、行业、场景、预算、部署、周期、POC筛选。

AI增强：AI根据需求推荐方案、生成采购建议。

权限：公开方案可读；详细报价需授权。

### 5.18 解决方案详情

目标：展示方案组成和采购路径。

功能：方案架构图、组成、适用客户、交付流程、验收指标、推荐组合、ROI、风险、POC。

AI增强：方案组件理解、组件替换、POC计划生成。

权限：方案详情公开；定制报价登录可见。

### 5.19 POC流程页

目标：管理POC全过程。

功能：需求确认、样例数据、测试、评审、采购沟通、测试指标、报告、待办、参与方。

AI增强：POC报告生成、待办提醒、测试集完整度检查、多供应商对比。

权限：POC数据严格隔离；AI访问样例数据需授权。

### 5.20 方案报价页

目标：结构化供应商报价。

功能：方案概述、报价明细、里程碑、验收指标、服务边界、附件、发送、草稿、PDF、智能审查。

AI增强：报价草稿、完整度审查、报价对比。

权限：报价私密；自动发送需供应商授权。

### 5.21 站内沟通页

目标：沉淀买卖双方上下文。

功能：会话列表、聊天窗口、关联需求、关联POC、关联方案、附件、AI回复建议、上下文面板。

AI增强：总结沟通、生成回复草稿、提取待办、同步到POC或报价。

权限：AI默认只读；自动回复需授权；AI生成消息打标。

## 6. 权限与审计

### 6.1 权限模型

采用 RBAC + ABAC + AI Delegation：

- RBAC：基于角色
- ABAC：基于组织、对象、状态、预算、风险等级
- AI Delegation：基于用户对AI的授权范围

### 6.2 必须记录的审计日志

- AI读取了什么内容
- AI代表谁执行动作
- AI使用了哪个授权策略
- AI生成了什么草稿
- AI提交了什么内容
- AI是否访问附件
- AI是否触发风控
- 用户是否确认
- 管理员是否干预

### 6.3 高风险动作

必须人工确认：

- 发布正式需求
- 提交正式报价
- 自动接单
- 访问敏感附件
- 导出客户联系方式
- 发起付款
- 生成合同
- 关闭POC
- 确认成交

## 7. MVP开发优先级

### MVP 0.1：传统平台闭环

首页、AI产品列表、产品详情、发布需求、需求大厅、需求详情、站内沟通、供应商后台、需求方控制台。

### MVP 0.2：AI可读平台

JSON-LD、Markdown页面、llms.txt、公开产品API、公开方案API、公开公司API、AI访问日志。

### MVP 0.3：AI代理草稿模式

AI生成需求草稿、方案草稿、报价草稿、POC报告、用户审批机制。

### MVP 0.4：AI半自动执行

AI自动匹配需求、推荐产品、筛选线索、接单草稿、站内回复草稿、权限策略中心。

### MVP 1.0：AI自主协作市场

MCP服务、A2A Agent Card、外部Agent身份认证、AI自主提需、AI自主接单、AI行为审计、POC自动评测、结构化报价。
